确定了上传webshell的方向，下面考虑escapeshellarg和escapeshellcmd处理的绕过。在上面我们了解到

escapeshellarg — 把字符串转码为可以在 shell 命令里使用的参数


功能 ：escapeshellarg() 将给字符串增加一个单引号并且能引用或者转码任何已经存在的单引号，这样以确保能够直接将一个字符串传入 shell 函数，shell 函数包含 exec(), system() 执行运算符(反引号)


定义 ：string escapeshellarg ( string $arg )

简单来说，escapeshellarg是为了防止用户利用shell的一些技巧（如分号、反引号等），执行其他命令。

escapeshellcmd — shell 元字符转义


功能：escapeshellcmd() 对字符串中可能会欺骗 shell 命令执行任意命令的字符进行转义。 此函数保证用户输入的数据在传送到 exec() 或 system() 函数，或者 执行操作符 之前进行转义。


反斜线（\）会在以下字符之前插入： &#;反引号|?~<>^()[]{}$, \x0A 和 \xFF。 *’ 和 “ 仅在不配对儿的时候被转义。 在 Windows 平台上，所有这些字符以及 % 和 ! 字符都会被空格代替。


定义 ：string escapeshellcmd ( string $command)

而escapeshellcmd则是为了防止用户的输入逃逸出“参数值”的位置，变成一个“参数选项”。

我们在Linux环境下，对127.0.0.1' a b进行测试

可以得到其处理的过程：

  • 经过escapeshellarg处理，先对中间没有配对的单引号转义，变成\'，再用两对单引号，分别将左右两部分括起来，从而起到连接作用，变成'127.0.0.1'\'' a b'
  • 经过escapeshellcmd处理，其处理时会对\进行转义，变成\\，而最后b'的单引号被视为没有配对，因而也被转义，最后变成'127.0.0.1'\\'' a b\'
  • 最后若当作命令执行，由于中间的\\被解释为字符\而不再具有转义作用，所以后面的'没有被转义，而是与 a前面的'配对成了一个空白连接符。最终在命令中可以简化为127.0.0.1\ a b'
